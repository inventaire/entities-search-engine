Guess what? We have our own tailored Wikidata Search Engine!!

This is very much just an ElasticSearch instance fed with a Wikidata dump subset to match inventaire.io's needs, that is, mainly to feed inventaire's autocomplete fields with Wikidata entities already filtered to match the fields' needs.

show me the code!

You can directly access ElasticSearch API, but endpoints will be progressively whitelisted to match inventaire.io's needs:

  • /wikidata/humans/_search
  • /wikidata/genres/_search

This is primarly addressing the needs of inventaire.io but you are very welcome to have fun with it. If you find value in it, please consider a donation, and if you wish to make an heavy use of this web service, get in touch!
